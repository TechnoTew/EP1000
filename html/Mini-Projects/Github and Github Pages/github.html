Why GitHub?

GitHub is a version control system that makes it easy to track and document files, including files that have since been deleted or modified. Through tools such as GitHub desktop, the complexity of Git is reduced as we do not need to type git commands into the terminal.


Why GitHub Pages?

GitHub pages allows the deployment of a simple website from the GitHub repository directly, without having to configure much.

Step 2: Creating and commiting files into the GitHub repository

For creating and committing files into the GitHub repository, we will be using GitHub Desktop, which is a Graphical User Interface that helps us manage our git repository. You can download GitHub Desktop here.

Shown above is the interface of GitHub Desktop.
The red rectangle contains the files that are being edited, deleted, or added, as shown by the coloured boxes beside the file path.
The green and cyan buttons are the commit and push buttons respectively. Committing allows you to set which files have been completed and are ready to be deployed to the remote GitHub repository. Pushing sends the files across to the remote GitHub repository to be activated in the case of GitHub Pages.
The yellow box contains the active changes to the selected file in the red box. As seen from the picture, the current file being selected has a newline removed but 8 lines added.

Step 3: Setting up GitHub Repository to support GitHub Pages

To set up GitHub Pages, first navigate to the settings of your repository. From there, select the pages menu on the sidebar and set the source to main.
If your repository has an index.html file in the root directly of the main branch, you will be able to access the page immediately after.
